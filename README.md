# A Heartfelt Welcoming

If you're here, you've probably been invited to our **Pre-Alpha Developer Program**. We welcome thee! You’re helping shape the foundations of Crucible.

## The Crucible Concept

Crucible reimagines how software is built by separating app development into two layers:

* Developers contribute the **operational logic** that makes systems function. These contributions - called _resources_ — define services and how frontends interact with them. This is the engine that powers Crucible’s ecosystem.

* Product professionals handle the **composition stage**. They choose which backend services to integrate. Each service includes a set of frontend widgets that become available once the product professional selects that service. They can then drag and drop these widgets onto a canvas to compose user experiences — the logic is already built-in.

When a product professional pays to use a service, **Crucible shares part of that revenue** with the developer whose logic enables it. Developers build. Product professionals compose. Both benefit.

## The Tiered Model

Crucible features a **tiered model** to reward developer participation. As developers contribute, they earn **points** that raise their tier and unlock new rewards.

Each tier increases their **revenue share**, up to **75%** at the highest tier. The more a developer contributes to Crucible’s ecosystem, the more they earn from the services that depend on their work.

### 2. Scaffold a Widget

A **widget** is a Crucible resource that operates on the frontend. It combines logic and visual elements such as buttons and inputs, handling all interaction with the backend.

To scaffold a widget, open a terminal and type:

<pre>
```sh
crux init
```
</pre>

This creates a simple widget on the working directory. See other scaffolding options with `crux init --help`.

Once the widget is created, refer to the included `README` file for additional instructions. It explains how to test, build, package, and distribute your widget.
